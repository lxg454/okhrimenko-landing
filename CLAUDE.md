# Правила работы с проектом

## Структура файлов
- Каждая страница — отдельный HTML-файл (index.html, about.html и т.д.)
- Никаких внешних библиотек и фреймворков (no Bootstrap, no Tailwind, no jQuery)
- CSS и JS пишем прямо внутри HTML-файла через `<style>` и `<script>`
- Иконки — только SVG, встроенные в HTML

## Дизайн
- Фон: тёмный (`#0a0a0a` или близкий)
- Текст: светлый (`#e2e8f0` или белый)
- Акцентный цвет: оранжевый (основной — `#f97316`, hover — `#ea580c`)
- Адаптивная вёрстка обязательна: сайт должен корректно выглядеть на мобильных (от 320px)
- Используем `clamp()` для шрифтов и `flexbox`/`grid` для раскладки

## Стиль текста
- Простой и дружеский язык, без канцелярита
- Короткие предложения, живые формулировки
- Обращение на «ты»

## Рабочий процесс
- Перед созданием нового файла — сначала описать, что планируется сделать, и дождаться подтверждения
- Перед большими изменениями в существующем файле — кратко обозначить, что именно меняется
- После изменений — коммитить в git с понятным сообщением на русском языке
